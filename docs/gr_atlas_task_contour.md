# Контур задач GR: Atlas + GR Hub

**Статус:** согласованное направление. Это план пилота, а не перенос данных: исходный GR-портал и рабочие данные пока не меняются.

## Решение

- **Atlas** — единственный источник задачи: исполнитель, срок, статус, приёмка и история изменений.
- **GR Hub** — рабочий экран GR: очередь по проектам, этапы, внешние ожидания, ссылки на документы и решения.
- GR Hub не хранит отдельную копию статуса задачи. Любое создание, изменение или приёмка задачи происходят в Atlas и сразу возвращаются в карточку GR Hub.
- Переход будет одним: до согласованной даты источник задач — действующий GR-портал; после проверки пилота — Atlas. Двойной записи не будет.

## Временный доступ для пилота

Пилот запускается без персональных учётных записей: один общий вход только для команды GR и только к 12 пилотным проектам.

- В истории действия отмечаются как «рабочая сессия GR» с датой и временем. Это не подтверждает, кто именно сделал изменение.
- Исполнитель и принимающий остаются рабочими полями задачи, а не подтверждённой личностью, выполнившей действие в системе.
- Оригиналы документов, персональные данные, суммы, договоры и юридические файлы в Hub не загружаются; остаются только рабочие карточки и ссылки.
- Через пилот нельзя проводить оплату, юридическое согласование, выдачу доступа или создавать обязательства.
- Если для действия понадобится персональная ответственность, пилот по этой задаче останавливается до появления персональных учётных записей.

## Что входит в одну задачу Atlas

| Блок | Что фиксируем |
|---|---|
| Смысл | название, проект, этап запуска, направление работы |
| Ответственность | один исполнитель, постановщик, при необходимости принимающий результат |
| Срок | срок или явная отметка «срок не назначен» |
| Статус | не начато · в работе · ждём · на приёмке · завершено · отменено |
| Ожидание | кого ждём, владелец следующего касания, дата следующего касания |
| История | кто и когда поставил, изменил, вернул или принял задачу |
| Связи | идентификатор проекта, ссылка на решение и ссылка на рабочую карточку GR Hub |

Для задач с приёмкой исполнитель передаёт результат в статус «на приёмке»; принимающий либо завершает, либо возвращает с комментарием. Для простых задач приёмка может быть отключена при постановке.

## Что остаётся в GR Hub

| В GR Hub | Зачем |
|---|---|
| Маршрут проекта по шести этапам | Понимать, где находится запуск |
| Очередь GR на сегодня | Видеть ближайшее действие и просроченные касания |
| Внешние ожидания и блокеры | Не терять, от кого зависит следующий шаг |
| Ссылки на документы | Работать с существующими папками без копирования файлов |
| Решения | Видеть вопрос, участников, протокол и связанные задачи |

Карточка GR Hub показывает задачу Atlas по её номеру и не позволяет существовать двум независимым версиям одной задачи.

## Пилотный набор

В пилот входят 12 проектов и 22 открытые задачи из действующего GR-портала. До переноса для каждой задачи проверяются:

1. Один исполнитель и проект.
2. Тип приёмки — нужен или не нужен.
3. Для «ждём» — кого ждём, владелец касания и следующая дата.
4. Для Dion и Amaia — юридическое лицо остаётся «требует уточнения», без догадок.
5. Даты не придумываются: если срок отсутствует в источнике, он остаётся неназначенным.

## Порядок запуска

1. Создать минимальное ядро задач Atlas с общим доступом только к 12 проектам и журналом «рабочая сессия GR».
2. Подключить к нему экран задач и карточки проектов в GR Hub.
3. Загрузить и проверить сначала две задачи: одну «в работе» и одну «ждём».
4. Проверить, что изменение, возврат на доработку и приёмка видны одинаково в Atlas и GR Hub.
5. После отдельного подтверждения на живую запись перенести оставшиеся 20 задач и назначить дату переключения источника.

## Критерии пилота

- У каждой открытой задачи один исполнитель и одна история.
- Ни одна задача не существует только в GR Hub.
- Ожидание без следующего касания не попадает в рабочую очередь.
- Команда GR за минуту видит по каждому проекту: следующее действие, ответственного, блокер и документ.
- В конце недели можно сравнить число зависших ожиданий и время поиска следующего действия с текущим порталом.

## Условия до живого переноса

1. Назначен ответственный за работу Atlas: доступы, резервные копии и восстановление.
2. Общий вход ограничен 12 пилотными проектами и не открывает финансовые, юридические или кадровые данные.
3. Есть единые идентификаторы проектов и людей для 12 пилотных карточек.
4. Определена дата переключения: после неё Atlas — единственный источник задач, а старый портал остаётся только архивом для этой выборки.
5. При необходимости персональной ответственности общий пилот прекращается, а соответствующая задача ждёт персональных учётных записей.

## Граница первого этапа

Не входят: интеграции с Bitrix24, 1С и Telegram; загрузка оригиналов документов; автоматические напоминания; перенос всех 121 задач. Это отдельные этапы после результата пилота.
