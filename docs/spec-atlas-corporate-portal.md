## Задача

Сейчас `index.html` (корень atlas.atamuragroup.kz) — чистый шлюз «вход по роли»: герой ATLAS + карточки 7 ролей + «Карта системы» + плашка-девиз. Задача — превратить `index.html` в корпоративный портал: дружелюбную домашнюю страницу с новостями, KPI-снапшотом, быстрыми ссылками и орг-инфо, чтобы человек видел актуальный контекст компании и конкретные действия, а не только меню выбора роли.

## 1. Маршрутизация (IA)

- `index.html` — новый корпоративный портал (описан в разделе 2).
- `role.html` — новый файл, точная копия ТЕКУЩЕГО `index.html` (весь HTML/CSS/JS 1:1, без изменений контента). `<title>` остаётся «ATLAS · строительная ERP · вход по роли».
- Рейл-навигация на всех 21 экране **не меняется**: пункт «Вход» (первый в рейле, `href="index.html"`) остаётся как есть везде и продолжает вести на `index.html` — теперь это портал. Никакого нового пункта в рейле не добавляется.
- Переход к выбору роли — через контентную ссылку внутри портала (плитка «Войти по роли →» → `role.html`), не через рейл.

## 2. Контент и раскладка портала (index.html)

Сверху вниз, в рамках существующего `<main>` (тот же рейл, тот же `.shell`):

1. **Приветствие** — заменяет прежний полноразмерный ATLAS-герой. Статичный заголовок «С возвращением» (без логики по времени суток, без персонализации по имени — демо-мокап без бэкенда/логина) + подпись «ATLAS · корпоративный портал ATAMŪRA» + дата. Вордмарк ATLAS остаётся, но меньше — как бренд-акцент, не главный элемент экрана.
2. **«На вас ждёт»** (action-items) — карточка со списком 3 демо-пунктов, каждый: цветной статус-маркер + заголовок + подзаголовок (объект/деталь) + ссылка-переход. Примеры пунктов и целевые экраны:
   - Акты на согласование → `docs.html`
   - Просроченный фронт ГПР → `gpr.html`
   - Заявки на закуп ждут визы → `snab.html#zakup`
   Данные — статичный массив в JS, редактируется вручную правкой файла.
3. **Ряд «новости + KPI»**:
   - Слева — **новости/объявления**: 2-3 демо-карточки (рубрика/дата, заголовок, короткий текст, ссылка «Читать →»). Ссылка ведёт на релевантный существующий экран, если рубрика к нему привязана (напр. новость про снабжение → `snab.html`); если релевантного экрана нет — некликабельная демо-плашка без перехода (тот же паттерн, что уже используется в ATLAS для demo-чипов без реального адресата). Статичный массив в JS.
   - Справа — **KPI-снапшот**: те же 4 показателя, что раньше были в hero-pills (ЖК / блоки / работы / просрочка), **тот же живой fetch** `gpr-engine/gpr_data.json` (код переносится из текущего `index.html` как есть, без изменений логики).
4. **Быстрые ссылки на модули** — ряд плиток: ГПР, Снабжение, Финансы, Документы, Лига + отдельная выделенная плитка **«Войти по роли →»** → `role.html` (это единственный путь к экрану выбора роли, кроме рейла).
5. **Орг-инфо/справочник** — нижняя строка с цифрами (9 ЖК · 18 подразделений) и раскрывающимся по клику списком подразделений (то же самое, `<details>` или JS-toggle, без новой страницы и без внешних ссылок).

Раскладка — «Вариант 1: задачи прежде новостей» (утверждён визуально в брейнсторминге): action-items идёт **над** рядом новости+KPI.

## 3. Данные, код, дизайн-система

- Дизайн-токены (navy/teal/gold/cream, тени, radius, sidebar-w), шрифты (Onest основной, Cormorant на акцентах — бренд-марка и приветствие), грейн-фон, структура `.shell`/`aside.nav`/`main` — 1:1 копия из текущего `index.html`/`dashboard.html`. Рейл (`aside.nav`) переносится в новый `index.html` без изменений (все 7 пунктов, включая «Вход» первым).
- KPI-fetch логика (`fetch('gpr-engine/gpr_data.json')`, `build()`, `render()`) переносится из старого `index.html` без изменений.
- Новости и action-items — статичные JS-массивы, зашитые в HTML-файл. Никакого бэкенда, никакой формы редактирования — правка контента = правка файла.
- `role.html` — байт-в-байт копия текущего `index.html` (весь контент: hero+snap-pills+роли+карта системы+creed footer), только заголовок остаётся как сейчас.

## 4. Вне рамок (explicitly out of scope)

- Изменение рейла на других 20 экранах — не трогаем.
- Реальный источник новостей (CMS, Bitrix24, Telegram-канал) — не подключаем, только демо-контент.
- Персонализация по логину/роли на портале — не делаем (публичный демо-мокап, входа с паролем нет).
- Новая отдельная страница-справочник подразделений — orq-инфо остаётся раскрывающимся блоком на самой странице, не новым экраном.
- Действия из «На вас ждёт» — это ссылки-переходы на существующие экраны, не интерактивная обработка (согласовать/отклонить прямо на портале не делаем).

## Приёмка

- `index.html` открывается и показывает портал (приветствие, «На вас ждёт», новости+KPI, быстрые ссылки, орг-инфо); KPI-цифры подтягиваются из `gpr-engine/gpr_data.json` как раньше.
- `role.html` открывается и показывает прежний контент входа по роли 1:1 (визуально и по функциональности не отличим от старого `index.html`).
- Переход «Войти по роли →» с портала ведёт на `role.html`; переходы из карточек «На вас ждёт» ведут на нужные существующие экраны.
- Рейл на всех 21 экранах не сломан, пункт «Вход» по-прежнему ведёт на `index.html` (портал).
- Консоль браузера чистая на обеих страницах, `node --check` (если применимо) проходит, вёрстка не съезжает на узких экранах (проверка как и на других экранах ATLAS — до ~720px).
