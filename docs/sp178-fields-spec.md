# sp178 — структурные поля для контроля снабжения по поставщику/договору

Чтобы автоматически считать «закуплено/привезено vs договор», ловить **перелимит**,
**повторные оплаты** и **не поданные** заявки — заявкам воронки «СЗ на закуп для стройки»
(#178/4) нужны СТРУКТУРНЫЕ поля. Замер 2026-06-07 (800 заявок): поставщик-поле 0%,
сумма `opportunity` 31%, договор-лимит и материал — нет. Эвристика по заголовку даёт ~5%
с дублями написаний («Альпиева» / «Альпиева А.С.» / «Альпиева АС» = один поставщик) —
для контроля денег это негодно. Нужны поля + дисциплина заполнения.

## Поля (добавить в смарт-процесс sp178)
| Поле | Тип Bitrix | Зачем |
|---|---|---|
| **Поставщик** | привязка к справочнику (Компания/список), НЕ текст | один поставщик = одна запись; накопительно по поставщику |
| **Договор** | привязка/строка (№ договора) | группировка заявок по договору |
| **Лимит по договору, ₸** | число (деньги) | «закуплено/привезено vs лимит» → сигнал перелимита |
| **Сумма заявки, ₸** | число (`opportunity` уже есть — ЗАПОЛНЯТЬ у всех) | накопление по поставщику/договору |
| **Материал / номенклатура** | справочник или строка | сшивка с планом ГПР + анти-дубль |
| **work_uid** (опц.) | строка | мост «материал → срок работы», точная сшивка с `/zakup` |

## Правила заполнения (важнее самих полей)
- **Поставщик** — только из справочника (иначе один поставщик размножается написаниями).
- **Сумма** — обязательна к стадии «Согласование оплаты» (сейчас пусто у 69%).
- **Договор + лимит** — заполнять при первой заявке по договору.

## Что это даёт (автоматически, read-only, без записи в CRM)
- **Накопительно по поставщику/договору:** заведено / оплачено / привезено **vs лимит**.
- **Перелимит:** Σ заявок по договору > лимита → стоп-сигнал.
- **Дубли:** две заявки с тем же материалом/суммой/договором → флаг (анти-повторная-оплата).
- **«Не подано»:** работа ГПР стартует, материал нужен к дате, а заявки нет → сигнал НУ/подрядчику.

## Пока полей нет — что нужно в твоих файлах (отчёты/договора), чтобы собрать «vs договор» сразу
Любой Excel подстрою, но идеально — строки вида:
`Поставщик · № договора · Лимит по договору, ₸ · Закуплено, ₸ · Оплачено, ₸ · Привезено`.
Тогда `/suppliers` соберётся из файла напрямую (Bitrix-данные не нужны для накопительного).
