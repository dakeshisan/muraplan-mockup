# ATLAS · MuraPlan — мокапы экранов (Фаза 1)

Кликабельные HTML-прототипы системы отслеживания стройки **ATLAS / MuraPlan (Фаза 1)**:
ролевые экраны и план/факт ГПР. Чистый статичный HTML/CSS/JS — **без сборки и зависимостей**,
открывается двойным кликом. Публикуется на GitHub Pages.

**Live:** https://dakeshisan.github.io/muraplan-mockup/

---

## Экраны — что где

| Файл | Экран | Открыть |
|---|---|---|

## Бренд-тема (держать во всех экранах)

- **Шрифты:** Cormorant (дисплей, крупные числа, вордмарк) + Jost (UI/текст).
- **Палитра:** крем `#F7F5EE` · навы `#284157` · бирюза `#007484` · золото `#CFB372`; зелёный/красный — статусы (готово / отставание).
- **Mobile-first.** Все стили — в `:root` и `<style>` каждого файла (CSS-переменные).

## Данные

- ГПР-данные (пятна → разделы → работы, план/факт/отставание) лежат в объекте `D` внутри **`gpr.html`**.
  Сводки пятен в `nu.html` и `web.html` согласованы с этим объектом.
- KPI, коэффициенты, премии, суммы — **условные (демо)**. Реальные суммы премирования на публичный мокап не выносить.

## Как править и публиковать

1. Правишь нужный `.html` — всё инлайн в одном файле: разметка + `<style>` + `<script>`.
2. Локально — просто открой файл в браузере (сборка не нужна).
3. `git commit` → `git push` в ветку `main`.
4. GitHub Pages пересобирается **~1–2 мин**. Обновлять жёстко (Cmd/Ctrl + Shift + R), иначе видна старая версия из кэша.

Дорабатывать можно в любом редакторе или через AI-ассистента — проект самодостаточен.

## Принципы

- Упрощать **подачу**, не выкидывать функции.
- Единая бренд-тема во всех экранах.
- Числа условные; реальные суммы и персональные данные не публиковать.
