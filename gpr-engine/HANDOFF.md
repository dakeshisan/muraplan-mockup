## 3. ГЛАВНЫЙ ПРИНЦИП (не нарушать)
«Корректируется исходя из факта» = про **прогноз**, не про план:
- **План (baseline) ЗАФИКСИРОВАН.** Не подгонять под факт — иначе исчезнет отставание.
- **Корректируется прогноз + отставание** при каждом новом факте.
- **Перепланирование (новый baseline) — решение руководства** (директор/PM), не автомат. Движок предлагает — человек утверждает. (KPI-инвариант: план задаётся сверху.)

## 4. Модель данных
```
Портфель → Объект (ЖК) → Очередь (q) → Блок («пятно») → Раздел → Работа
```
Готовность катится снизу вверх. Метод %: взвешенно по длительности работ (как MS Project);
в прототипе — на уровне блока (avg). **«Пятно» = блок, «очередь» = q.**

## 5. Текущий прототип (что уже есть, LIVE)
- `index.html`: массив `OBJECTS` (4 ЖК / 38 блоков), у каждого блока `[имя, план%, факт%]` — **реальные данные отчёта 28.05.2026**.
- Движок: `gf()` (факт), `avg`, rollup по блоку/очереди/объекту/портфелю, отставание = план−факт, прогноз `при +RATE%/нед → 100% через N нед` (RATE=6, **условный темп**).
- UI: дерево объект→очередь→блок; **факт редактируется** (number-инпут) → готовность/отставание/прогноз пересчитываются мгновенно (`recompute()`); факт хранится в `localStorage` (`gpr_fact_v1`); «↺ сбросить факт».
- Проверено в браузере: правка факта блока → live-пересчёт портфеля + персист после reload, без ошибок.

## 6. Источник данных и как его читать
- **PDF-отчёты** `Отчет <ЖК> 28.05.2026.pdf` (в `~/Documents/Claude/gpr-source/ГПР по объектам/`): по каждому блоку шапка `Пятно "X" N очередь` + План%/Факт%, и **по-работная таблица** (37 работ: план%, факт%, начало, окончание, дни, отставание, договор/оплата/материалы, исполнитель).
- **Парсинг PDF:** установлен `pymupdf` (`import fitz`); `pdftotext`/`pdfplumber` — НЕТ. Сводка блока = строка слов, где есть `Пятно "X" N очередь` + два % (План, Факт). Имена в zip — UTF-8 без флага → чинить `name.encode('cp437').decode('utf-8')`.
- **.mpp (MS Project) — бинарные, без спецсофта не парсятся.** Их импорт (→ baseline) — отдельная задача Фазы 2 (нужна Java-библиотека MPXJ или экспорт в XML/Excel).
