# ATLAS · ГПР-движок (gpr-engine)

Отдельный проект: **ГПР как живой движок**, который пересчитывается исходя из факта.
Не «ещё один экран», а **единый источник** план/факт, над которым строятся все ролевые
экраны (портфель → объект → участок), бот и снабжение.

**Прототип (Фаза 1):** самодостаточный `index.html` (план + факт + расчёт в браузере,
факт сохраняется в localStorage). **Реальный продукт (Фаза 2):** ядро + БД факта + импорт
из MS Project + API. Логика расчёта одна и та же.

## Главный принцип (важно)

«Корректируется исходя из факта» — это про **прогноз**, не про план:

- **План (baseline) зафиксирован.** Если план автоматически подгонять под факт — исчезнет
  само отставание (всё всегда «в графике»). Baseline трогать нельзя.
- **Корректируется прогноз и отставание.** При каждом новом факте движок пересчитывает:
  готовность (план vs факт), отставание (п.п. и дни), критический путь, прогнозную дату сдачи.
- **Перепланирование (новый baseline) — решение руководства** (директор/PM), не автомат.
  Движок *предлагает* пересмотр, человек *утверждает*. (KPI-инвариант: план задаётся сверху,
  не занижается под лёгкое перевыполнение.)

## Модель данных

```
Портфель (компания)
 └─ Объект (ЖК)            Атмосфера / Аура / Керуен / Аксай резорт
     └─ Очередь (q)        2,3,4 …  ← «q» в MS Project
         └─ Блок (пятно)   3, 4, 5 / 25 А / 1.1 …  ← «пятно» = блок
             └─ Раздел     Общестрой / Отделка / Инженерка / Благоустройство
                 └─ Работа  37 работ: план%, факт%, сроки, отставание, исполнитель
```

Готовность катится снизу вверх: работа → раздел → блок → очередь → объект → портфель.
Метод %: взвешенно по длительности работ (как в MS Project). В прототипе — на уровне блока
(данные отчёта ГПР 28.05.2026); работный уровень — следующий слой (он есть в PDF-отчётах).

## Что считает движок (`engine`)

- **Готовность** план/факт на любом уровне (rollup).
- **Отставание** = план − факт (в п.п.; в реальном — в днях по критическому пути).
- **Критический путь** — что именно держит сдачу (раздел/работа с макс. отставанием).
- **Прогноз сдачи** — при текущем темпе: дата выхода на 100% и вердикт «успеваем / не успеваем».
- **What-if:** меняешь факт → весь портфель и прогноз пересчитываются мгновенно.

## Данные

Сид — из реальных отчётов ГПР на **28.05.2026** (4 ЖК, план/факт по блокам/очередям).
Источник: PDF-отчёты (`.mpp` MS Project — бинарные, парсятся отдельным импортёром в Фазе 2).

## Дорожная карта

1. ✅ **(прототип)** блок-уровень, факт редактируется и сохраняется, live-пересчёт.
2. ✅ **Работный уровень** — 1322 работы из PDF (`parse_gpr.py` → `gpr_data.json`),
   drill-down объект→блок→раздел→работа, **узкие места по отставанию в днях**
   (критический путь портфеля), what-if. Сверка: [`VALIDATION.md`](VALIDATION.md).
3. **Импортёр MS Project** (`.mpp` → baseline) или периодический экспорт.
4. **Настоящий сетевой критический путь (CPM)** по зависимостям задач из `.mpp`
   (PDF зависимостей не содержит) — сразу после п.3.
5. **Приём факта от бота «Флэш»** (НУ→РП→подтверждение) — реальный поток факта.
6. **API/данные**, которые читают мокап-экраны (портфель/объект/участок/РП) — единый источник.
7. **БД факта**, аудит-след, согласование перепланирования.
