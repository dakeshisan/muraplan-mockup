## Ключевые находки парсинга

1. **37 работ на блок — это ~39 строк.** В разделе слаботочки нумерация
   перезапускается: №27, №28 встречаются дважды с разными названиями
   (27 «Установка светильников» / 28 «Монтаж АПС», затем 27 «Видеонаблюдение» /
   28 «Домофоны»). **№ не уникален** → работы ключуются по позиции, не по №.
   У Аксая шаблон короче (~28–29 строк).
2. **Текст PDF не в визуальном порядке** → разбор по координатам (кластеризация слов
   по y, классификация колонок по x). Заголовки блоков различаются по объектам:
   `Пятно "3" 2 очередь` (Аура) · `Пятно "25 А" 5 очередь` (Атмосфера) ·
   `Пятно "1,1"- 1 очередь` (Керуен).
3. **На каждой работе:** план%, факт%, начало, окончание, длительность (дни),
   **отставание в днях**, у части — материалы/договор/оплата, исполнитель
   (ТОО Градо-А, ТОО Строй Тех Групп, ИП КазСтройБлок, ИП Дана…), статус договора.

## Сводимость блочных %: блок — authoritative, работы — детализация

Проверка: воспроизводит ли свод работ (взвешенно по календарным дням) блочные
План/Факт из шапки отчёта.

| | сред \|Δ\| | макс \|Δ\| | смещение |
|---|---|---|---|
| **Факт** (взвеш. по дням vs отчёт) | **3.1 пп** | 12 пп | почти всегда выше отчёта |
| **План** (взвеш. по дням vs отчёт) | **6.0 пп** | 15 пп | систематически выше отчёта |

**Вывод (заложен в движок):** блочные **План/Факт берутся из отчёта как
authoritative** (свод MS Project), из работ НЕ пересчитываются. Расхождение
систематическое и объяснимое: PDF даёт **календарную** длительность, а MS Project
взвешивает по **рабочим** дням / своему baseline (плюс план в колонке работ —
кумулятивный 100% к концу работы, а блочный «План» — time-phased на дату отчёта).

Работный уровень используется для трёх вещей, которые блочный % не даёт:
- **drill-down** (раздел → работа: сроки, исполнитель, материалы);
- **узкие места** по колонке «Отставание в днях» (что держит сдачу);
- **what-if**: правка факта работы двигает прогноз взвешенно по дням
  (baseline-привязка: блок = отчётный_факт + Σ(Δ работы × дни)/Σ дней).

## Оговорки

- **«Критический путь» здесь = рейтинг по отставанию в днях** из отчёта, не сетевой
  CPM. Настоящий CPM (по зависимостям задач) требует `.mpp` — отдельный пункт
  роадмапа сразу после импортёра MS Project.
- Темп прогноза (+6%/нед) условный — калибруется по факту / .mpp.
- Сроки поставки материалов и связка с фронтом — на стороне модуля «Снабжение».
