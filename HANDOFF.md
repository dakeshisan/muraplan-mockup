## 1. Что это
Кликабельные HTML-мокапы системы отслеживания стройки **ATLAS / MuraPlan (Фаза 1)**.
Чистый статичный HTML/CSS/JS, **без сборки** (открывается двойным кликом / на Pages).
Один продукт — разные экраны по роли. Иерархия:

```
Портфель (компания) → Объект (ЖК) → Участок (блок)   + роли + геймификация
```

## 7. Как править / смотреть / публиковать
- **Править:** один `.html`, всё инлайн (разметка+стиль+скрипт).
- **Смотреть локально:** `python3 -m http.server 8799 --bind 127.0.0.1` в папке репо → `http://127.0.0.1:8799/<файл>`.
- **Проверка через браузер:** Chrome MCP `navigate` на `127.0.0.1` (localhost не перехватывается, в отличие от github.com) → `javascript_tool` для проверки/`computer screenshot`. ГОЧА: screenshot иногда висит на `document_idle` (шрифты) — проверяй состояние через `javascript_tool`.
- **Коммит** (глобальной git-identity нет): `git -c user.name="..." -c user.email="..." commit -m "..."`.
- **Публикация:** push в `main` → Pages пересобирается ~1–2 мин. **Стандартное правило: публиковать после каждого существенного изменения.** Способ push через SSH-ключ — в памяти проекта (`muraplan-mockup-project.md`).
- **Проверка live:** `curl` страницы + `grep` маркеров (CDN-лаг до ~1 мин, разные файлы доезжают по очереди).
